# e057f6 slot 终端运行期历史人工验收清单

> 自动测试覆盖服务端 stream/降级/reset 数据链；本清单用于真浏览器人工验收。当前文件仅产出清单，未代表已执行。

## 1. 连续回看连接前与连接后历史

步骤：
1. 打开含 slot 终端的需求详情页，确认 claude/codex 任一 pane 已显示终端内容。
2. 在对应真实 pane 中先产生连接前历史，例如输出 200 行带编号文本。
3. 刷新浏览器或重新打开该需求详情页，让终端重新连接。
4. 连接后继续输出 300 行带编号文本。
5. 在一体化终端区域用鼠标滚轮向上连续回看，再向下回到底部。

预期：
- 能连续看到连接前历史尾部与连接后新输出。
- 不再只能看到当前一屏。
- 上滚查看历史时，新输出不强制把视口拉回底部；已经贴底时新输出继续跟随到底部。

## 2. claude/codex CLI 真实 TUI 观感

步骤：
1. 在 claude 或 codex pane 中进入真实 CLI/TUI 场景。
2. 观察普通输出、命令输入回显、全屏/alt-screen 界面切换。
3. 退出 TUI 后回看历史。

预期：
- 普通输出像真实终端一样进入历史。
- alt-screen 内容不进入 scrollback，这是符合终端语义的正确预期。
- 退出 alt-screen 后主屏历史仍可回看。

## 3. 06c62eb 回归：超高 pane 外滚与双底部贴底

步骤：
1. 打开左右 split 或较窄布局，让终端 pane 当前屏高度超过容器高度。
2. 用滚轮先查看当前屏上半/下半，再继续上滚进入 xterm 历史。
3. 回到底部后继续产生新输出。

预期：
- 纵向只出现一根可用滚动入口，不出现互相抢焦点的双滚动条。
- host 外滚能看到超高当前屏内容。
- 到 host 顶部后继续上滚能进入历史。
- xterm 历史与 host 外层均在底部时，新输出保持贴底。

## 4. PageUp 与触控历史入口

步骤：
1. 在有足够历史的终端内尝试 PageUp/PageDown。
2. 在触控板或触屏设备上用惯性滚动/滑动回看历史。
3. 重复测试在底部与非底部两种状态下的新输出行为。

预期：
- PageUp/PageDown 或触控入口可作为历史回看的可访问性补充。
- 触控滚动不会卡在只能看一屏的状态。
- 非底部查看历史时不被新输出强制打断。

## 5. 降级提示

步骤：
1. 在隔离测试 pane 或可安全操作的 pane 上预先占用 `pipe-pane` 槽位。
2. 打开对应 slot 终端页面。
3. 观察终端 header。
4. 释放 `pipe-pane` 槽位，等待自动重试升级。

预期：
- 槽位占用期间界面显示「历史受限(快照模式)」。
- 释放后自动升级到 stream 模式，提示消失。
- 升级后继续输出可进入运行期历史。
